ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ К ФОРМЕ РЕГИСТРАЦИИ

1. Общие положения
   1.1. Форма регистрации предназначена для создания нового аккаунта пользователя.
   1.2. Форма должна содержать следующие поля:
        - Электронная почта (email)
        - Логин (имя пользователя)
        - Пароль
        - Подтверждение пароля
        - Чекбокс согласия на обработку персональных данных

2. Поле "Электронная почта"
   2.1. Поле обязательно для заполнения.
   2.2. Значение должно соответствовать формату email (например: user@example.com).
   2.3. Email не должен быть уже зарегистрирован в системе.
   2.4. Регистр символов в email игнорируется (user@Example.COM = user@example.com).

3. Поле "Логин"
   3.1. Поле обязательно для заполнения.
   3.2. Длина логина — от 3 до 20 символов.
   3.3. Логин может содержать только латинские буквы, цифры и символы подчёркивания (_).
   3.4. Логин должен быть уникальным (не использоваться другим пользователем).

4. Поле "Пароль"
   4.1. Поле обязательно для заполнения.
   4.2. Минимальная длина — 6 символов.
   4.3. Пароль должен содержать:
        - как минимум одну заглавную латинскую букву,
        - как минимум одну строчную латинскую букву,
        - как минимум одну цифру,
        - как минимум один специальный символ (!@#$%^&* и т.п.).
   4.4. Пароль не должен совпадать с логином или email.

5. Поле "Подтверждение пароля"
   5.1. Поле обязательно для заполнения.
   5.2. Значение должно точно совпадать со значением поля "Пароль".

6. Согласие на обработку персональных данных
   6.1. Чекбокс "Я согласен на обработку персональных данных" должен быть отмечен.
   6.2. Без отметки регистрация невозможна.

7. Кнопка "Зарегистрироваться"
   7.1. Активна только при корректном заполнении всех полей и отметке согласия.
   7.2. При нажатии отправляются данные на сервер для создания аккаунта.
   7.3. После успешной регистрации пользователь перенаправляется на страницу подтверждения email или личного кабинета.

8. Обработка ошибок
   8.1. При некорректном вводе в любом поле отображается понятное сообщение об ошибке рядом с этим полем.
   8.2. Все поля сохраняют введённые значения после ошибки (кроме паролей — по усмотрению безопасности).

9. Безопасность
   9.1. Пароли передаются и хранятся в зашифрованном виде (хэширование).
   9.2. После трёх неудачных попыток регистрации с одного IP может применяться временная блокировка или CAPTCHA.

10. Доступность
    10.1. Форма должна быть доступна через клавиатуру и поддерживать screen readers.
    10.2. Все поля имеют подписи (<label>) и атрибуты доступности.
